实现了一个基于链表页式内存池（Linked-List Memory Slab）的录制数据管理系统。核心思想是利用 VirtualAlloc 分配固定大小（4KB）的内存页来存储连续的录制事件，并通过链表管理这些页，同时维护一个索引表（IndexMap）以便快速随机访问。
以下是每个函数的详细流程简述：

前提条件:传入的 RecordItemLinkedList 结构体必须通过动态内存分配（即位于堆中，函数 malloc, calloc, realloc），不可为栈上局部变量。（不听劝计算机就哭给你看！）

1. 核心数据操作函数

void* FormatRecordItem(RecordItem Item, void* DeviceInfo)
功能：将离散的录制项头信息和设备数据打包成一块连续的堆内存。
流程：
计算大小：根据 Item.Type（鼠标、键盘、剪贴板、截图），计算 DeviceInfo 中变长数据（如按键字符串、剪贴板内容）的实际长度，得出总大小 TotalSize。
分配内存：使用 calloc 分配 sizeof(RecordItem) + TotalSize 的内存，并清零。
拷贝头部：将 RecordItem 结构体拷贝到内存块起始位置。
拷贝数据：将 DeviceInfo 的基础部分拷贝到头部之后。
重定位变长数据：如果是键盘或剪贴板类型，计算变长字符串在新区块中的相对偏移地址，更新 StringInfo.pData 指针，并将实际字符串数据拷贝到该位置。
返回：返回指向这块连续内存的指针。

bool AppendRecordItem(RecordItemLinkedList& MemorySlab, void* Item, RecordIndexMapping& IndexMap);	
功能：将格式化好的记录项追加到内存链表（Slab）末尾。
流程：
查找尾节点：遍历链表找到最后一个有数据的节点（或头节点）。
初始化检查：如果当前节点未分配内存页（pBuffer 为空），调用 VirtualAlloc 分配 4KB 内存。
空间检查：判断 当前游标 + 新项大小 是否超过 4KB。
若溢出：创建新链表节点，分配新的 4KB 内存页，链接到当前节点后，切换当前指针到新节点。
指针重定位（关键）：如果是键盘/剪贴板类型，将该项内部 StringInfo.pData 指针修改为相对于当前新内存页基址的绝对地址（因为写入后指针必须指向有效内存）。
写入数据：将数据拷贝到 pBuffer + BufferCursor 位置。
更新索引：将新项的起始地址存入 IndexMap，更新 BufferCursor。

bool DeleteRecordItem(RecordItemLinkedList& MemorySlab, RecordIndexMapping& IndexMap, int Index)
功能：删除指定索引的记录项，并整理内存碎片。
流程：
定位页：通过 IndexMap 获取待删除项的地址，利用掩码 (PAGE_BASE_MASK) 提取页基址，在链表中找到对应的节点。
计算移动量：计算待删除项的大小。
内存前移：如果待删除项不是页内最后一项，使用 memmove 将其后的所有数据向前覆盖移动，填补空缺。
清零尾部：将移动后末尾多余的旧数据清零。
更新索引：
从 IndexMap 中移除被删项的条目。
遍历后续同页的索引项，将其地址减去删除项的大小（因为它们都前移了）。
更新游标：减少当前页的 BufferCursor。

bool ClearRecordItem(RecordItemLinkedList& MemorySlab)
功能：清空所有录制数据，释放内存。
流程：
遍历链表：从头到尾遍历所有节点。
释放资源：对每个节点的 pBuffer 调用 VirtualFree 释放内存页，使用 free 释放链表节点本身。
重置头节点：将头节点的指针、游标、Next 指针全部重置为初始状态。

bool ModifyRecordItem(RecordItemLinkedList& MemorySlab, RecordItem* NewItem, RecordIndexMapping& IndexMap, int Index)
功能：修改指定索引的记录项（支持变长修改）。
流程：
大小比较：比较新旧记录项的总大小。
若相等：直接 memcpy 覆盖，结束。
若不等：进入复杂的重排逻辑。
重排逻辑：
定位到所在页。
情况A（新数据更小）：直接在原地写入新数据，将后方数据前移，多余空间清零。
情况B（新数据更大）：
若当前页剩余空间足够：后方数据后移（需确保不溢出，代码逻辑似乎通过插入新页或紧凑处理来解决，具体看 CompactSlab 调用）。
注：当前代码逻辑在变大时似乎尝试插入新节点或依赖后续的 CompactSlab 来整理，逻辑较为复杂，涉及数据搬移和索引更新。
整理：调用 CompactSlab 确保内存连续性。

bool ForEachRecordItem(RecordItemLinkedList& MemorySlab, RecordCallBack cbfn)
功能：遍历所有记录项并执行回调函数。
流程：
双重循环：外层循环遍历链表节点（Slab），内层循环遍历节点内的数据块（通过 Offset 递增）。
解析项：根据偏移量计算 RecordItem 头和 DeviceInfo 数据指针。
执行回调：调用 cbfn(Item, DeviceInfo)。
中断机制：如果回调返回 false，立即停止遍历并返回 false。

2. 回调函数 (Callbacks)
static bool PlayRecordItemCallBack(RecordItem* Item, void* DeviceInfo)
功能：回放录制的动作。
流程：
类型分发：根据 Item->Type 进入不同分支。
执行动作：
鼠标：调用 Device.MouseSend 模拟移动/点击。
键盘：切换输入法（如需）、模拟按键输入（支持 Shift/Ctrl 热键）。
剪贴板：根据操作码（Set/Get/Paste/Clear）和字符集（ANSI/Unicode）操作系统剪贴板。
截图：(TODO) 预留接口。
延迟：执行 Sleep(Item->delay) 模拟录制时的时间间隔。

static bool DisplayRecordItemCallBack(RecordItem* Item, void* DeviceInfo)
功能：将记录项信息显示在 UI 列表控件 (lvw_RecordQueue) 上。
流程：
插入行：在列表视图新增一行。
填充基础信息：填入序号、延迟时间、数据大小。
格式化详情：根据类型解析数据，生成人类可读的描述字符串（如“绝对坐标(100,200)”、“输入字符:Hello”等）。
更新UI：将描述字符串填入列表的详情列。

static bool RelocateStringInfoCallBack(RecordItem* Item, void* DeviceInfo)
功能：文件读取后专用，修复因内存地址变化而失效的字符串指针。
流程：
检查类型：仅处理键盘和剪贴板数据。
基址比对：比较 RecordItem 所在的页基址 与 StringInfo.pData 指向的页基址。
重计算指针：如果基址不同（说明是从文件读入到了新内存页），提取 pData 原有的页内偏移量 (PAGE_OFFSET_MASK)，加上当前新的页基址，重新赋值给 pData。

static bool BuildIndexMapCallBack(RecordItem* Item, void* DeviceInfo)
功能：重建索引映射表。
流程：
清空：清空全局 RecordBaseMap。
添加：将当前遍历到的项指针加入映射表。（通常配合 ForEachRecordItem 使用，用于初始化索引）。

3. 文件 I/O 函数

bool ValidRecordFile(const wchar_t* Path)
功能：验证文件格式是否合法。
流程：
读文件头：检查 FileMagic ("MKCP") 和版本号。
遍历页头：循环读取 PageHead。
检查 PageMagic ("PG")。
根据 PageSize 跳过数据块。
完整性检查：确保文件指针最终严格停留在文件末尾，无多余垃圾数据。

bool WriteRecordFile(const wchar_t* Path, RecordItemLinkedList Buffer)
功能：将内存中的数据写入磁盘文件。
流程：
写文件头：写入 "MKCP" 标志和版本。
遍历链表：对每个非空节点：
构造 PageHead (标志 "PG" + 有效数据长度 BufferCursor)。
写入 PageHead。
写入该节点的有效数据块 (pBuffer 到 BufferCursor)。

bool ReadRecordFile(const wchar_t* Path, RecordItemLinkedList& Buffer)
功能：从磁盘文件加载数据到内存。
流程：
验签：读取并验证文件头。
循环读页：
读取 PageHead，验证 "PG" 标志。
分配内存：为当前页调用 VirtualAlloc 分配 4KB。
读取数据：将文件中的数据块直接读入分配的内存页。
创建节点：创建新的链表节点连接，准备读取下一页。
后续处理：读取完成后，通常需要调用 RelocateStringInfoCallBack 来修复指针（虽然此函数内未显式调用，但这是标准流程的下一步）。

DWORD GetRecordFileSize(const wchar_t* Path) & bool FileExists(const wchar_t* Path)
功能：辅助函数，分别获取文件大小和判断文件是否存在。
流程：调用 Windows API (GetFileSize, GetFileAttributes) 实现。

void SelectRecordFile()
功能：弹出文件选择对话框。
流程：实例化 CFileDialog，设置过滤器 (.rd)，显示对话框，若用户确认则更新文本框路径。(注：代码中 delete SelectFileDlg 是错误的，栈对象不需要 delete，且 MFC 对话框通常是栈分配)*。

4. 内存优化与工具函数

bool CompactSlab(RecordItemLinkedList& MemorySlab, RecordIndexMapping& IndexMap);
功能：内存碎片整理（压缩）。
流程：
创建新池：创建一个空的临时链表 NewMemorySlab。
重放写入：遍历旧链表中的所有记录项，依次调用 AppendRecordItem 将它们写入新池。由于 Append 会自动紧凑排列，这消除了删除操作留下的空洞。
替换：清空旧链表，将新链表的数据结构拷贝回原变量。
重建索引：在 Append 过程中会重新生成 IndexMap。

float GetMemorySlabUilization(RecordItemLinkedList MemorySlab) & CString DisplayUilization(RecordItemLinkedList MemorySlab)
功能：计算并格式化显示内存利用率。
流程：
统计：遍历链表，累加 BufferCursor (已用字节) 和 固定页大小 (0x1000 * 页数)。
计算：(已用 / 总共) * 100。
格式化：调用 FormatBytes 将字节转换为 KB/MB 单位，生成 "已用/总计 (百分比)" 字符串。

CString FormatBytes(DWORD Size)
功能：将字节数转换为人类可读的单位 (B, KB, MB, GB)。
流程：循环除以 1024 直到数值小于 1024，根据循环次数选择单位，格式化输出。

代码亮点与潜在风险提示
指针重定位技术：代码巧妙地利用了 VirtualAlloc 的 64KB 对齐特性和 4KB 页大小，通过保存页内偏移量来实现文件持久化后的指针恢复 (RelocateStringInfoCallBack)。这是处理包含指针的结构体序列化的经典高级技巧。
内存安全：大量使用了 calloc 清零和边界检查，防止脏数据。
潜在风险：
SelectRecordFile 中 delete SelectFileDlg 会导致崩溃，因为 CFileDialog 是栈对象。
ModifyRecordItem 在处理变长数据扩容时逻辑非常复杂，如果当前页空间不足且后续逻辑未正确处理跨页移动，可能会导致数据截断或越界（目前看它依赖 CompactSlab 兜底，但中间过程可能存在短暂的不一致）。
ReadRecordFile 读取后必须手动调用 ForEachRecordItem(..., RelocateStringInfoCallBack)，否则程序访问字符串指针时会崩溃。当前代码未在 ReadRecordFile 内部自动调用这一点，需要调用者注意。
